OkHttp

使用

​ 首先是创建一个OkHttpClient实例，然后创建一个Request实例，调用newCall和enqueue发起请求并获得结果。

Request

每一个HTTP请求包含一个URL、一个方法（GET或POST或其他）、一些HTTP头。请求还可能包含一个特定内容类型的数据类的主体部分。

Response

响应是对请求的回复，包含状态码、HTTP头和主体部分。

Call

OkHttp使用Call抽象出一个满足请求的模型，尽管中间可能会有多个请求或响应。执行Call有两种方式，同步或异步。

请求流程

OkHttpClient#Builder()获得一个OkHttpClient实例。

拼接一个我们需要的Request。

OkHttpClient#newCall(): 实际上生成了一个RealCall对象。

RealCall#enqueue(): 调用了Dispatcher#enqueue()。

Dispatcher：Dispatcher采取单例线程池,线程空闲时，只能保留60秒。

Dispatcher#enqueue()：采用了3个deque存放call，先进先出，优先放置于runningAsyncCalls，但是runningAsyncCalls只能存放最多maxRequests（64）个请求，所以不能进入时，就放置于readyAsyncCalls。当该请求出队列执行时，调用RealCall#execute()。

RealCall#execute()：核心是调用了RealCall#getResponseWithInterceptorChain()。

RealCall#getResponseWithInterceptorChain()：采用责任链方式，添加多个拦截器

RealCall#AsynsCall#execute():执行监听回调。
